# Majlis Atfal-ul-Ahmadiyya Ghana Management System

## Project Overview

Build a modern, mobile-first management system for Majlis Atfal-ul-Ahmadiyya Ghana.

The system should provide complete member management, location management, analytics, reporting, and automated lifecycle management for Atfal members across Ghana.

This is not a financial management system. The primary focus is member records, administration, growth tracking, and analytics.

The application should feel modern, fast, clean, and professional, similar to Notion, Linear, Stripe Dashboard, Airtable, or Apple's design philosophy.

---

# Core Objectives

The platform must allow:

* Full CRUD operations for members
* Location hierarchy management
* Member search and filtering
* Automatic age calculation
* Automatic birthday updates
* Automatic wing classification
* Automatic graduation between wings
* Graduation alerts for Khuddam transitions
* National, Regional, Zonal, Circuit, and Jama'at analytics
* Excel and PDF exports
* Mobile-first responsive experience
* Offline-friendly data entry where possible

---

# Membership Structure

The system manages only three classifications.

## Atfal Sughir

Age:
7–11 years

## Atfal Kabir

Age:
12–14 years

## Khuddam

Age:
15 years and above

---

# Automatic Age Engine

Age must never be manually entered.

The system calculates age from Date of Birth.

Every day:

* Recalculate member age
* Update age badges
* Update wing classification

Example:

DOB: 10 June 2014

Current Date: 10 June 2026

Age: 12

Wing:
Atfal Kabir

---

# Automatic Graduation Rules

The system automatically moves members between wings.

## Sughir → Kabir

When member turns 12:

Automatically change wing from:

Atfal Sughir

to

Atfal Kabir

No notification required.

---

## Kabir → Khuddam

When member turns 15:

Automatically change wing from:

Atfal Kabir

to

Khuddam

Create system notification.

Create graduation record.

Display in graduation dashboard.

National, Regional, and Zonal administrators should see graduation alerts.

* First Name
* Middle Name
* Last Name
* Date of Birth
* Profile Photo

Automatically Display:

* Age
* Wing Classification

---

## Step 2 — Administrative Information

Fields:

* Sector
* Region
* Zone
* Circuit
* Jama'at
* Position

Sector, Region, and Zone are dropdowns.

Circuit and Jama'at are smart searchable inputs.

Example:

User types:

"Ahi"

Suggestions appear:

* Ahinsan
* Ahinsan Estate
* Ahinsan Central

If no match exists:

Create new Circuit or Jama'at automatically.

Future users can reuse the newly created value.
