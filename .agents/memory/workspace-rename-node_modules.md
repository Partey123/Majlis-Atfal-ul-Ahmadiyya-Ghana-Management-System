---
name: Workspace rename node_modules
description: What to do when a pnpm workspace package directory is renamed/moved.
---

After renaming a pnpm workspace package directory (e.g. `artifacts/atfal-ghana` → `frontend`), the hoisted symlinks inside `<package>/node_modules/` become stale and point to non-existent paths. Running `pnpm install` alone does NOT fix this because pnpm sees the lockfile as up to date.

**Rule:** After any workspace directory rename, run `rm -rf <package>/node_modules && pnpm install` for every renamed package.

**Why:** pnpm hoists packages via symlinks anchored to the original path. The new directory gets an empty or partially-linked `node_modules/` that passes `ls` checks but fails at runtime (e.g. `@vitejs/plugin-react` directory exists but `package.json` is missing).

**How to apply:** Any time a `mv` is done on a workspace package folder, always follow it with a `rm -rf <package>/node_modules && pnpm install` before attempting to build or start.
