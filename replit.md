# Atfal-GMS — Majlis Atfal-ul-Ahmadiyya Ghana Management System

A national membership management platform for Majlis Atfal-ul-Ahmadiyya Ghana, tracking members across Atfal Sughir, Atfal Kabir, and Khuddam wings.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 3000)
- `pnpm --filter @workspace/atfal-ghana run dev` — run the frontend (port 5000)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/db run migrate` — run migrations

Required env:
- `DATABASE_URL` — Postgres connection string
- `JWT_SECRET` — secret for signing JWTs (defaults to insecure dev value; **must be set in production**)
- `ADMIN_USERNAME` — login username (default: `admin`)
- `ADMIN_PASSWORD` — login password (default: `admin123`)

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- Auth: JWT in HTTP-only cookie (`atfal_session`), 24h expiry
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)
- Frontend: React 19, Vite 7, Tailwind CSS v4, wouter, TanStack Query, shadcn/ui

## Architecture decisions

- **Auth is render-guard only**: `AppContent` in `App.tsx` checks `useAuth()` and conditionally renders `<Login />` vs the full app. No redirect-based routing for auth.
- **Wing boundaries**: Atfal Sughir < 12, Atfal Kabir 12–14, Khuddam ≥ 15. These are applied server-side in `computeWing()` in `members.ts` (source of truth) and mirrored in the frontend's `useWingCalc.ts` and `MemberWizard.tsx`.
- **Lazy loading**: All route-level pages are lazy-loaded with `React.lazy` + `Suspense` to keep the initial bundle small.
- **Member creation uses a DB transaction**: The POST /members route wraps insert + history + usage-count increments in a single transaction so partial failures don't corrupt the DB.
- **Analytics by-sector uses a single GROUP BY**: Instead of 3 separate queries (one per sector), a single `GROUP BY sector` query with `count(*) filter (where wing = '...')` expressions handles it.

## Product

The app manages the full lifecycle of Atfal members in Ghana:
- **Dashboard**: real-time stats — total members, upcoming graduations, wing breakdown
- **Member list**: searchable/filterable table with wing, sector, circuit, jamaat filters
- **Member profile**: full detail view with history timeline
- **Member wizard**: step-by-step guided member creation with DOB→wing auto-assignment
- **Graduations**: upcoming and recent graduation tracking
- **Analytics**: sector-by-sector and region breakdown charts
- **Locations**: manage circuits and jamaats

## Gotchas

- After renaming workspace packages, run `rm -rf <package>/node_modules && pnpm install` to rebuild symlinks — stale hoisted packages will silently break builds.
- The backend's `dev` script runs `pnpm run build` (esbuild) then `pnpm run start` — it is **not** a watch server. After source changes, the workflow must be restarted to rebuild.
- `backend/tsconfig.json` and `frontend/tsconfig.json` both extend `../tsconfig.base.json` (one level up, at workspace root). Don't accidentally use `../../`.
- Health check (`GET /api/healthz`) is explicitly exempt from auth in `requireAuth.ts` by path check.
- Auth cookie name is `atfal_session`. Use `credentials: "include"` in all frontend fetch calls to the API.
